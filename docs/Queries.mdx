# GraphQL Queries

Learn how to generate GraphQL queries from Zod schemas using `zod2gql`.

## Basic Query Generation

The simplest way to generate a GraphQL query is using the `createQuery` helper function:

<Source
  language="typescript"
  dark
  format={true}
  code={`
import { z } from 'zod';
import { createQuery } from 'zod2gql';

const userSchema = z.object({
  id: z.string(),
  name: z.string(),
  email: z.string(),
});

const query = createQuery(userSchema, {
  operationName: 'GetUser',
  variables: { id: '123' }
});

console.log(query);
/* Output:
query GetUser($id: String!) {
  user(id: $id) {
    id
    name
    email
  }
}
*/
  `}
/>

## Query with Nested Objects

`zod2gql` recursively processes nested object schemas:

<Source
  language="typescript"
  dark
  format={true}
  code={`
const addressSchema = z.object({
  street: z.string(),
  city: z.string(),
  zipCode: z.string(),
});

const userSchema = z.object({
  id: z.string(),
  name: z.string(),
  address: addressSchema,
});

const query = createQuery(userSchema, {
  operationName: 'GetUser',
  variables: { id: '123' }
});

console.log(query);
/* Output:
query GetUser($id: String!) {
  user(id: $id) {
    id
    name
    address {
      street
      city
      zipCode
    }
  }
}
*/
  `}
/>

## Query with Arrays

Arrays in Zod schemas are also properly handled:

<Source
  language="typescript"
  dark
  format={true}
  code={`
const postSchema = z.object({
  id: z.string(),
  title: z.string(),
  content: z.string(),
  tags: z.array(z.string()),
  comments: z.array(z.object({
    id: z.string(),
    text: z.string(),
    author: z.string(),
  })),
});

const query = createQuery(postSchema, {
  operationName: 'GetPost',
  variables: { id: 'post123' }
});

console.log(query);
/* Output:
query GetPost($id: String!) {
  post(id: $id) {
    id
    title
    content
    tags
    comments {
      id
      text
      author
    }
  }
}
*/
  `}
/>

## Query with Optional Fields

Optional fields in Zod schemas are included in the query:

<Source
  language="typescript"
  dark
  format={true}
  code={`
const userSchema = z.object({
  id: z.string(),
  name: z.string(),
  email: z.string(),
  profilePicture: z.string().optional(),
  bio: z.string().optional(),
});

const query = createQuery(userSchema, {
  operationName: 'GetUser',
  variables: { id: '123' }
});

console.log(query);
/* Output:
query GetUser($id: String!) {
  user(id: $id) {
    id
    name
    email
    profilePicture
    bio
  }
}
*/
  `}
/>

## Query with Circular References

Circular references are handled automatically thanks to Zod's `z.lazy()`:

<Source
  language="typescript"
  dark
  format={true}
  code={`
// Define schemas with circular references
const categorySchema = z.object({
  id: z.string(),
  name: z.string(),
  description: z.string().optional(),
});

// Add circular reference using z.lazy()
const productSchema = z.object({
  id: z.string(),
  name: z.string(),
  category: categorySchema,
  relatedProducts: z.array(z.lazy(() => productSchema)),
});

const query = createQuery(productSchema, {
  operationName: 'GetProduct',
  variables: { id: 'product123' }
});

console.log(query);
/* Output:
query GetProduct($id: String!) {
  product(id: $id) {
    id
    name
    category {
      id
      name
      description
    }
    relatedProducts {
      id
      name
      category {
        id
        name
        description
      }
      relatedProducts
    }
  }
}
*/
  `}
/>

## Controlling Recursion Depth

You can limit the depth of recursion to prevent overly complex queries:

<Source
  language="typescript"
  dark
  format={true}
  code={`
const query = createQuery(productSchema, {
  operationName: 'GetProduct',
  variables: { id: 'product123' },
  maxDepth: 3 // Limit recursion depth
});

console.log(query);
/* Output:
query GetProduct($id: String!) {
  product(id: $id) {
    id
    name
    category {
      id
      name
      description
    }
    relatedProducts {
      id
      name
      category
      relatedProducts
    }
  }
}
*/
  `}
/>
